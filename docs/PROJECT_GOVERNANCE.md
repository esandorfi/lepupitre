# Project Governance

This file is the source of truth for maintainer roles used in release and code-signing governance.

## Maintainer
- Emmanuel Sandorfi ([`@esandorfi`](https://github.com/esandorfi))

## Maintainer responsibilities
- Product and architecture direction.
- Release ownership and quality gates.
- Security triage and disclosure coordination.
- Signing/notarization approval ownership until additional approvers are onboarded.

## Signing roles
- Committers/reviewers: Le Pupitre maintainers (currently Emmanuel Sandorfi).
- Signing approver: Emmanuel Sandorfi.

## Security baseline for maintainers
- MFA must be enabled on GitHub accounts with write/admin access.
- MFA must be enabled on any signing service account (SignPath, Apple, etc.).
